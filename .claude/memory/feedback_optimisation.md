**Règle :** Ne jamais rendre quelque chose qui n'est pas 100% optimisé. Viser l'excellence technique et l'optimisation maximale dans tout ce qui est produit.

**Why:** L'utilisateur a dit : "Ne me rendez jamais quelque chose qui n'est pas 100% optimisé. Je veux toujours que ça soit le plus optimisé possible."

**How to apply:**
- Ne pas se contenter d'une solution "qui marche" : chercher la meilleure solution
- Optimiser performances, lisibilité, maintenabilité
- Éliminer le code mort, les abstractions inutiles, les hacks temporaires
- Proposer des améliorations même si non demandées explicitement
- Rester pragmatique : l'optimisation ne doit pas devenir de la sur-ingénierie pour des cas hypothétiques
